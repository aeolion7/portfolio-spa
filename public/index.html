BundesData
An application that displays data about the Bundesliga, the premier soccer league in Germany.
BundesData

BundesData is a web application that retrieves soccer data using the user-requested parameters and displays it in an easily understandable and visual format. The user must select which season's data they would like to view, and then, by selecting a matchday and clicking the button, the application will display all of the games that took place on that particular matchday, including team information and logos, half-time and full-time scores, when the match took place (or will take place), and the name (if known) of the head referee who officiated the match.

The underlying idea came about when I decided I wanted to build a functional application relating to a personal interest. In developing BundesData, I solidified a lot of the useful skills in application development such as consuming an API to present data. The user interface is built in React and state management is done through Redux.

Development of new features is ongoing!

Yesteryear
A mock e-commerce site that sells 90s memorabilia. Tamagotchis, Gameboy Colors, Beanie Babies, and more!
Yesteryear

Yesteryear is an e-commerce site that is aimed at appeasing 1990s nostalgia. The site features login and signup functionality via both standard means and Google OAuth, and features a cart and checkout system, for both registered users and guests.

Yesteryear was built using Node.js. It is a single-page application featuring a front-end built in React and uses Redux in order to manage state and make it accessible throughout the entire site. It uses an API built and authenticated using Express.js, and interacts with a PostgreSQL database full of user information, product information, and order histories and present carts using Sequelize.

In developing Yesteryear over the course of a week with a team of three others, I strengthened my understanding of RESTful APIs, authentication, proper Git workflow, and Agile methodolgies.

BabelLens
An application that allows you to take a picture of text with your phone and translate it into the langauge of your choice.
BabelLens

BabelLens is a mobile application centered around language. The main functionality of the application is in the ability to take a picture of text with your phone's camera and have it translated into the language of your choice. Built in React Native using the Expo toolchain, the idea for BabelLens came out of a personal interest in language. Having studied different languages over the years, I often frustratingly notice a word in an advertisement for which I could not remember the translation in another language, and I tasked myself with building an application able to deal with this problem.

In addition to the picture-to-translation functionality, I also included a text translator that allows the user to input whatever text they like instead of having to walk around and try to find an image that contains the word they would like to translate. The 10 most recent translation requests are stored on application state, and there are options that allow the user to select the languages they would like, in addition to being able to request the scanned text before translation so that they can verify its accuracy.

Working as a solo developer over the course of 5 days, building BabelLens afforded me the opportunity to familiarize myself with React Native, experiment with computer vision, and strengthen my understanding of utilizing external APIs to perform functions and collect data.

ChromeVision
A Google Chrome extension and React SPA combination that provides sentiment analysis and visualizes a user's emotional data.
ChromeVision

ChromeVision is a Google Chrome extension and accompanying single page application that provides data visualization based on emotional sentiment analysis in correlation to web browsing activity. The Chrome extension requests access to the user's webcam, and then sends their image data to the Microsoft Face API for analysis. The response contains data correlating to the user's facial expressions, which is subsequently visualized in easily-readable charts and graphs to display their emotional trends while browsing the web.

In developing ChromeVision, there were some interesting challenges. Since a Chrome extension lives within the browser, we chose to develop it using HTML, CSS, and JavaScript in order to keep it lightweight. This posed additional challenges in that the Microsoft Face API accepts one of only two options: an image URL or binary data for processing. We as a team did not want to store user snapshots in any persistent manner, and, as such, we could not use an image URL. We were able to solve this problem by converting the webcam snapshots directly into binary data using a byte array.
