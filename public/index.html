BundesData

BundesData is a web application that retrieves soccer data using the user-requested parameters and displays it in an easily understandable and visual format. The user must select which season's data they would like to view, and then, by selecting a matchday and clicking the button, the application will display all of the games that took place on that particular matchday, including team information and logos, half-time and full-time scores, when the match took place (or will take place), and the name (if known) of the head referee who officiated the match.

The underlying idea came about when I decided I wanted to build a functional application relating to a personal interest. In developing BundesData, I solidified a lot of the useful skills in application development such as consuming an API to present data. The user interface is built in React and state management is done through Redux.

Development of new features is ongoing!

A mock e-commerce site that sells 90s memorabilia. Tamagotchis, Gameboy Colors, Beanie Babies, and more!
Yesteryear

Yesteryear is an e-commerce site that is aimed at appeasing 1990s nostalgia. The site features login and signup functionality via both standard means and Google OAuth, and features a cart and checkout system, for both registered users and guests.

Yesteryear was built using Node.js. It is a single-page application featuring a front-end built in React and uses Redux in order to manage state and make it accessible throughout the entire site. It uses an API built and authenticated using Express.js, and interacts with a PostgreSQL database full of user information, product information, and order histories and present carts using Sequelize.

In developing Yesteryear over the course of a week with a team of three others, I strengthened my understanding of RESTful APIs, authentication, proper Git workflow, and Agile methodolgies.
